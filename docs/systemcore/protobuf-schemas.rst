Protobuf Schemas
=================

All complex Dscomm topics use **Protobuf-encoded binary blobs** (NT4 type code 5 / raw). The Protobuf package is ``mrc.proto``.

Control Messages
-----------------

ProtobufControlData
^^^^^^^^^^^^^^^^^^^

The primary DS -> Robot control message. Replaces the old 8-byte UDP control packet.

+---------------+-------------------------------+--------------+-----------------------------------+
| Field         | Protobuf Type                 | Field Number | Description                       |
+===============+===============================+==============+===================================+
| MatchTime     | int32                         | 2            | Match countdown in seconds        |
+---------------+-------------------------------+--------------+-----------------------------------+
| Joysticks     | repeated ProtobufJoystickData | 3            | Up to 6 joystick states           |
+---------------+-------------------------------+--------------+-----------------------------------+
| CurrentOpMode | ProtobufOpMode                | 4            | Currently selected operation mode |
+---------------+-------------------------------+--------------+-----------------------------------+
| ControlWord   | uint32                        | 5            | Control state bitfield            |
+---------------+-------------------------------+--------------+-----------------------------------+
| GameData      | string                        | 6            | FMS game-specific message         |
+---------------+-------------------------------+--------------+-----------------------------------+

Wire Format
-----------

These Protobuf messages are serialized using standard Protobuf binary encoding, then wrapped as NT4 type code 5 (raw/binary) values in a MsgPack binary frame::

  NT4 Binary Frame (MsgPack):
    [topic_id, timestamp, 5, <protobuf_encoded_bytes>]

The MsgPack layer sees an opaque binary blob. The Protobuf decoding happens at the application layer.
